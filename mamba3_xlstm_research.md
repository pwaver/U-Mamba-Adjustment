# Mamba-2 vs xLSTM: Architecture Comparison and Hybridization Research

## Background

This document captures research comparing the xLSTM architecture
([Beck et al. 2024, arxiv 2405.04517](https://arxiv.org/abs/2405.04517))
with Mamba-2 / State Space Duality
([Dao & Gu 2024, arxiv 2405.21060](https://arxiv.org/abs/2405.21060)),
in the context of this project's use of Mamba2 as the SSM layer ("mamba-3") in UMambaBot3 and UMambaEnc3.

---

## Core Mathematical Correspondence

Both are forms of **masked linear recurrent attention**. The mLSTM matrix memory
`C_t ∈ ℝ^{d×d}` is mathematically identical to Mamba-2's SSM hidden state
`h_t ∈ ℝ^{N×P}` with N=P=d. The covariance update rule:

```
C_t = f_t C_{t-1} + i_t v_t k_t^T    (mLSTM)
h_t = a_t h_{t-1} + B_t x_t          (Mamba-2)
```

are both outer-product accumulations into a matrix state — the same underlying operation.

## Part 1: xLSTM Architecture

### 1.1 sLSTM — Scalar Memory with Exponential Gating

sLSTM keeps a scalar cell state but replaces sigmoid gates with exponential gates,
plus a normalizer state to prevent unbounded growth.

**Forward pass:**
```
c_t = f_t * c_{t-1} + i_t * z_t       (cell state)
n_t = f_t * n_{t-1} + i_t             (normalizer state)
h_t = o_t * (c_t / n_t)               (hidden state)

i_t = exp(ĩ_t)                         (INPUT GATE: exponential — can exceed 1)
f_t = σ(f̃_t)  or  exp(f̃_t)           (FORGET GATE: sigmoid or exp)
o_t = σ(õ_t)                           (output gate, sigmoid)
```

With exponential gating, `f_t` can exceed 1 (when `f̃_t > 0`), enabling
**hard overwrite semantics**: when `i_t >> f_t * c_{t-1}`, old values are
effectively erased. This is what allows sLSTM to solve tasks like Nearest
Neighbor Search and parity that standard LSTM fails at.

**Stabilization** (prevents float32 overflow):
```
m_t = max(log(f_t) + m_{t-1}, log(i_t))
i'_t = exp(ĩ_t - m_t)
f'_t = exp(log(f_t) + m_{t-1} - m_t)
```
Analogous to the log-sum-exp trick in softmax. The ratio `i'_t / f'_t` is
identical to the unstabilized ratio — only the absolute scale changes.

**Memory mixing:** sLSTM retains hidden-to-hidden recurrent connections
(`R_z, R_i, R_f, R_o ∈ ℝ^{d×d}`), mixing states across cells within a head.
This is what gives sLSTM higher expressiveness — and what makes it
non-parallelizable (see §3 below).

### 1.2 mLSTM — Matrix Memory with Covariance Update Rule

mLSTM replaces the scalar cell state with a matrix `C_t ∈ ℝ^{d×d}`, using
explicit Q/K/V projections. No hidden-to-hidden connections.

**Recurrent form:**
```
C_t = f_t * C_{t-1} + i_t * (v_t k_t^T)     (d×d matrix state)
n_t = f_t * n_{t-1} + i_t * k_t              (d-dim normalizer vector)
h_t = o_t ⊙ (C_t q_t / max(|n_t^T q_t|, 1)) (output)

q_t = W_q x_t    (query)
k_t = W_k x_t / √d  (key, scaled)
v_t = W_v x_t    (value)
i_t = exp(ĩ_t)   (scalar input gate, exponential)
f_t = σ(f̃_t) or exp(f̃_t)  (scalar forget gate)
o_t = σ(õ_t)     (vector output gate)
```

The normalizer `n_t^T q_t` approximates the total expected inner product of
past keys with the current query — a principled content-aware denominator.
The `max(·, 1)` prevents division by near-zero.

**Parallel (training) form:**

Define the gate mask matrix `D ∈ ℝ^{T×T}` (lower triangular, causal):
```
D_{ij} = F_{ij} · exp(ĩ_j)
where F_{ij} = prod_{k=j+1}^{i} f_k    (cumulative forget product)
```

Stabilized:
```
D_log = log(F) + Ĩ
D' = exp(D_log - row_max(D_log))
```

Full parallel forward:
```
H_tilde = ((Q K^T / √d) ⊙ D' / n) · V
H = σ(Õ) ⊙ H_tilde
```

This is **masked linear attention** with a data-dependent, two-gate mask.

## Part 2: Mamba-2 / State Space Duality

### 2.1 The SSD Duality

For scalar-identity SSMs (`A_t = a_t · I`), the output can be written as masked
attention:

```
M_{ji} = a_{j:i} · (C_j^T B_i) = L_{ji} · G_{ji}
where L = 1SS(a),   G_{ji} = C_j^T B_i
```

Therefore `M = L ⊙ (C B^T)` — masked kernel attention with mask
`L = 1SS(a)` (1-semiseparable matrix generated by scalar decay chain),
queries `Q=C`, keys `K=B`, values `V=X`.

**1-SS mask:** `L_{ij} = a_j · a_{j-1} · ... · a_{i+1}` — a scalar cumulative
product, one scalar per timestep per head. Simpler and less expressive than
mLSTM's two-gate mask, but admits a highly efficient block algorithm.

### 2.2 The SSD Block Algorithm

Instead of full recurrence (O(TN²), not matmul-friendly) or full attention
(O(T²N), memory-expensive), SSD uses block decomposition:

- **Diagonal blocks:** Quadratic attention within chunks — small dense matmuls,
  parallelizable across chunks.
- **Off-diagonal blocks:** Factored as `[C-factors] · [A-factors] · [B-factors]^T`
  — inter-chunk state propagation via short 1-SS scan, then matmuls.

Total: O(TN²) FLOPs, O(TN) memory, dominated by matmuls
(hardware-friendly on GPU tensor cores).

### 2.3 Mamba-2 Architecture Changes vs Mamba-1

1. **Parallel projections:** `x, z, A, B, C = parallel_proj(u)` — all from same
   input simultaneously (vs. sequential in Mamba-1). Enables tensor parallelism
   with 1 all-reduce per block.
2. **GroupNorm/RMSNorm** before final projection for training stability.
3. **Larger head dimension** P = 64–128 (vs. P=1 in Mamba-1) — larger recurrent
   state without proportional compute cost.
4. **Scalar A:** `A_t = a_t · I` per head — simpler than Mamba-1's diagonal A,
   enables the SSD duality.

## Part 3: Comparison and Hybridization

### 3.1 The Key Difference: Gate Structure on the Mask

| | mLSTM | Mamba-2 |
|---|---|---|
| Attention mask `L` | `F_{ij} · exp(ĩ_j)` — **two gates** | `a_j · ... · a_{i+1}` — **one scalar** |
| Forget gate `f_t` | `σ(·)` or `exp(·)`, can exceed 1 → hard overwrite | implicit in `a_t` |
| Input gate `i_t` | `exp(ĩ_t)` — explicit, separate | no explicit input gate |
| Normalizer | vector `n_t`, content-aware denominator | none (GroupNorm post-hoc) |
| QKV projections | explicit W_q, W_k, W_v | B (≅K), C (≅Q), X (≅V) |

**Critical insight:** mLSTM's mask is the Hadamard product of a 1-SS forget
matrix and a diagonal input gate matrix:
```
D_{ij} = [1SS(f)]_{ij} · [diag(exp(ĩ))]_{jj}
```
A diagonal times a 1-SS matrix is still 1-SS. Therefore **mLSTM's mask is 1-SS**,
and the SSD block algorithm applies to mLSTM without modification. The two
architectures are not in conflict — mLSTM is a strict superset of Mamba-2's
parameterization.

### 3.2 Expressiveness

mLSTM and Mamba-2 are in the same formal language class: both are 1-SS masked
linear attention, expressible as bounded-rank semiseparable matrix operations.
Neither can solve state tracking problems (parity, modular arithmetic) in the
formal language-theoretic sense.

**sLSTM with memory mixing** is strictly more expressive — its hidden-to-hidden
connections push it outside the 1-SS class. But this is precisely what makes it
non-parallelizable with the SSD algorithm.

The xLSTM solution: interleave sLSTM blocks (expressiveness) with mLSTM blocks
(capacity + parallelism) in `xLSTM[a:b]` ratio.

### 3.3 Hybridization Pathways

All three below are compatible with the existing SSD algorithm (no kernel changes needed):

#### Tier 1 — Low effort: Two-gate A + normalizer

Replace Mamba-2's single `a_t` with separate forget and input gates:
```python
# Current Mamba-2
a_t = proj_a(u)          # scalar per head

# Proposed hybrid
f_t = torch.sigmoid(proj_f(u))   # forget gate
i_t = torch.exp(proj_i(u))       # input gate (exponential)
a_t = f_t * i_t                  # combined — still 1-SS mask
```

Add normalizer state:
```python
# Alongside h_t = a_t * h_{t-1} + B_t * x_t:
n_t = f_t * n_{t-1} + i_t * B_t
y_t = C_t.T @ h_t / max(abs(C_t.T @ n_t), 1)
```

Cost: two extra small projections per block, one extra vector recurrence O(N).
Gains: hard overwrite semantics, principled content-aware normalization.

#### Tier 2 — Medium: Full mLSTM gating in SSD block

The mLSTM recurrent form with SSD parallel training:
```
h_t = f_t * h_{t-1} + i_t * B_t x_t    (vs Mamba-2: h_t = a_t h_{t-1} + B_t x_t)
y_t = C_t^T h_t / normalizer_t
```
Mathematically identical to mLSTM but trained with SSD block algorithm —
2–8× faster than naive scan. No change to SSD kernel; only A/gate parameterization changes.

#### Tier 3 — Architecture: xLSTM[a:b]-style interleaving

Mix sLSTM-based blocks (for state tracking, serial) with SSD/Mamba-2 blocks
(for capacity and parallel training). The ratio controls the expressiveness/throughput
tradeoff. The xLSTM codebase implements this directly.

### 3.4 What Cannot Be Incorporated

sLSTM's **hidden-to-hidden memory mixing** cannot be included in an SSD-based model
without abandoning parallel training. It is architecturally incompatible with the
1-SS semiseparable structure required by the SSD block algorithm.

## Summary for This Project

`UMambaBot3` and `UMambaEnc3` use `Mamba2` from `mamba_ssm` as their SSM layer.
The mLSTM gates (Tier 1 above) are **drop-in compatible** and represent the most
promising near-term experiment:

- Modify `MambaLayer3.__init__` to pass separate `f_t` and `i_t` projections
  rather than a single `a_t`
- Add normalizer state to `MambaLayer3.forward`
- No changes to the SSD CUDA kernel or the rest of the network

This would create a `MambaLayer3Gated` variant testable against the current
`MambaLayer3` baseline with a new trainer class following the same pattern
as `nnUNetTrainerUMambaBot3` / `nnUNetTrainerUMambaEnc3`.
